# herkeuren-app

## Description
Deze app is geschreven in JAVA tijdens de module "Ontwerpen en Programmeren" an de NCOI.
We moesten een simpele app maken met minimaal 4 klassen en het moest op je werk bruikbaar zijn.

Samenvatting van de module-opdracht:
De Herkeuring applicatie is gemaakt om als hulpmiddel te dienen bij het herkeuren van (te) lang staande groente, zodat je makkelijk en eenduidig een oordeel krijgt. 
Later zouden er andere vragen toegevoegd kunnen worden en eerdere beoordelingen zouden meegenomen kunnen worden.
De applicatie bestaat uit de volgende klassen: Herkeuring, Groente, Vraag, Antwoord, Resultaat. Herkeuring is de Hoofdklasse.

Deze klassen zijn verwerkt in een UML klassendiagram en daarna omgezet naar Java programmeercode. Tijdens de ontwikkeling is het klassendiagram meerdere keren aangepast, even als de attributen en de methodes tijdens het programmeren. 
Ook de punten die bij de antwoorden horen die voor het resultaat zorgen, heb ik meermaals aangepast, omdat sommige dingen zwaarder wegen dan andere dingen.
Uit de uitgevoerde Error guessing- test is gebleken dat de applicatie naar behoren werkt. Aan de hand van de vragen krijg je een oordeel en vanuit dat oordeel kun je de groente weggooien of laten verwerken.

Reflectie:

Het was voor mij lastig om een applicatie te verzinnen. Ik heb veel gekeken op internet naar ideeën en aan de keurmeesters gevraagd wat zij zouden willen. Toen kwam ik hierop. 
Eerst had ik het idee om leveranciers te beoordelen naar aanleiding van wat vragen, maar de kwaliteit van de groente is per periode wisselend vanwege het weer en dat ligt bijna nooit aan de leverancier, dus toen kwam ik op dit idee.

Deze module- opdracht gaf me wat meer inzicht in het object georiënteerd programmeren. Ik had hier al mee kennis gemaakt door kleine spelletjes te maken en een kalender. 
Wat wel nieuw voor mij was, was een klassendiagram maken en daarna te beginnen met programmeren. 

Ik heb vaak het diagram aangepast, omdat het toch iets anders moest met programmeren dan ik van te voren had verwacht. Ik denk dat dat komt, omdat ik er nog niet veel ervaring mee heb.
Ik had wel een probleem met het draaien van deze applicatie in Netbeans, terwijl deze gewoon werkte in Eclipse waarin ik deze applicatie heb geschreven. Door voor Netbeans updates te installeren, werkte het daarna ook in Netbeans.
